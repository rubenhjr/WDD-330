Chapter 2 programming basics

Scope: Using const and let to declare variables means they are block scoped, so their value only exists inside the block they are declared in.
Global Scope: Any variable declared outside of a block is said to have global scope. This means it is accessible everywhere in the program.
Local Scope In ES6, blocks can be used to create alocal scope. This means that any variables defined inside a block using the let or const will only be available inside that block and not be accessible outside of that block. This is known as having local scope, as the variable is only visible in the locality of the block.
Template Literals: are a special types of string that were introduced in ES6. Template literals use the backtick character, `.
This has the advantage of being able to use both types of quote mark within the string.

Chapter 3 Arrays:

An array is an ordered list of values. To create an array literal, simply write a pair of square brackets:
"const myArray = [];" We can create an array literal using square brackets that already contain some initial values, so thereâ€™s no need to add each value one by one. Here's an example:
const avengers = ['Captain America', 'Iron Man', 'Thor', 'Hulk']; The concat() method can be used to merge an array with one or more arrays:
avengers.concat(['Hulk','Hawkeye', 'Black Widow']); The join() method can be used to turn the array into a string that comprises all the items in the array, separated by commas:
avengers.join();
Results: 'Captain America, Iron Man, Thor, Hulk, Hawkeye, Black Widow'
Or you can choose a separator other than a comma by placing it inside the parentheses:
avengers.join(' & ');
'Captain America & Iron Man & Thor & Hulk & Hawkeye & Black Widow'

Chapter 4:

To define a function literal we can use a function declaration:
function hello(){console.log('Hello World!');}
Another way of defining a function literal is to create a function expression. This assigns anonymous function a variable:
const goodbye = function(){ console.log('Goodbye World!'); };
Arrow Function: These make declaring functions much more succinct by using less verbose syntax.
Arrow functions have a number of advantages over other ways of declaring functions:
They are much less verbose than normal function declarations.
Single parameters don't need putting into parentheses.
The body of the function doesn't need placing inside a block if it's only one line.
The return keyword isn't required if the return statement is the only statement in the body of the function.
They don't bind their own value of this to the function.
